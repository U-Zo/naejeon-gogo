## ADDED Requirements

### Requirement: 멤버 등록
시스템은 새 멤버를 등록할 수 있어야 한다. 멤버 등록 시 이름, 주 포지션, 가능 포지션 목록을 필수로 입력해야 한다. 초기 MMR은 1000으로 설정된다.

#### Scenario: 고정 멤버 등록
- **WHEN** 운영자가 이름 "Player1", 주 포지션 "mid", 가능 포지션 ["mid", "top"]을 입력하고 등록 버튼을 누른다
- **THEN** 시스템은 해당 정보와 초기 MMR 1000으로 멤버를 생성하고 멤버 목록에 표시한다

#### Scenario: 임시 참가자 등록
- **WHEN** 운영자가 "임시 참가자" 옵션을 선택하고 이름과 포지션 정보를 입력한다
- **THEN** 시스템은 임시 플래그가 설정된 멤버를 생성한다

#### Scenario: 필수 정보 누락
- **WHEN** 운영자가 이름 없이 등록을 시도한다
- **THEN** 시스템은 필수 정보 입력을 요구하는 오류 메시지를 표시한다

### Requirement: 멤버 정보 수정
시스템은 등록된 멤버의 이름, 주 포지션, 가능 포지션, MMR을 수정할 수 있어야 한다.

#### Scenario: 포지션 변경
- **WHEN** 운영자가 멤버의 주 포지션을 "mid"에서 "jungle"로 변경한다
- **THEN** 시스템은 해당 멤버의 주 포지션을 업데이트하고 저장한다

#### Scenario: MMR 수동 조정
- **WHEN** 운영자가 멤버의 MMR을 직접 수정한다
- **THEN** 시스템은 입력된 MMR 값으로 업데이트한다

### Requirement: 멤버 삭제
시스템은 멤버를 삭제할 수 있어야 한다. 삭제 시 확인 절차를 거쳐야 한다.

#### Scenario: 멤버 삭제 확인
- **WHEN** 운영자가 멤버 삭제 버튼을 누른다
- **THEN** 시스템은 삭제 확인 다이얼로그를 표시한다

#### Scenario: 삭제 확정
- **WHEN** 운영자가 삭제 확인 다이얼로그에서 확정한다
- **THEN** 시스템은 해당 멤버를 목록에서 제거한다. 기존 전적 기록은 유지된다.

### Requirement: 멤버 목록 조회
시스템은 등록된 모든 멤버를 목록으로 표시해야 한다. 이름, 주 포지션, MMR, 전적(승/패)이 표시되어야 한다.

#### Scenario: 멤버 목록 표시
- **WHEN** 운영자가 멤버관리 탭을 연다
- **THEN** 시스템은 모든 멤버를 이름, 주 포지션, MMR, 승/패 전적과 함께 목록으로 표시한다

#### Scenario: 임시 참가자 구분
- **WHEN** 멤버 목록에 임시 참가자가 포함되어 있다
- **THEN** 시스템은 임시 참가자를 시각적으로 구분하여 표시한다

### Requirement: 포지션 관리
멤버는 반드시 하나의 주 포지션을 가져야 하며, 0개 이상의 가능 포지션을 추가로 지정할 수 있다. 포지션 종류는 top, jungle, mid, adc, support 5가지이다.

#### Scenario: 주 포지션 필수
- **WHEN** 운영자가 주 포지션 없이 멤버를 등록하려 한다
- **THEN** 시스템은 주 포지션 선택을 요구한다

#### Scenario: 가능 포지션 복수 선택
- **WHEN** 운영자가 가능 포지션으로 "top", "mid", "support"를 선택한다
- **THEN** 시스템은 3개의 가능 포지션을 저장한다
