## ADDED Requirements

### Requirement: 참가자 선택
시스템은 매칭을 위해 멤버 풀에서 정확히 10명의 참가자를 선택할 수 있어야 한다.

#### Scenario: 10명 참가자 선택
- **WHEN** 운영자가 멤버 목록에서 10명을 체크하고 매칭 시작을 누른다
- **THEN** 시스템은 선택된 10명으로 매칭 알고리즘을 실행한다

#### Scenario: 10명 미만 선택
- **WHEN** 운영자가 10명 미만을 선택하고 매칭 시작을 누른다
- **THEN** 시스템은 정확히 10명을 선택해야 한다는 메시지를 표시한다

#### Scenario: 10명 초과 선택
- **WHEN** 운영자가 10명 초과를 선택하고 매칭 시작을 누른다
- **THEN** 시스템은 정확히 10명을 선택해야 한다는 메시지를 표시한다

### Requirement: 포지션 기반 팀 배정
시스템은 각 팀에 top, jungle, mid, adc, support 5개 포지션을 배정해야 한다. 참가자의 주 포지션과 가능 포지션을 고려하여 배정한다.

#### Scenario: 주 포지션 우선 배정
- **WHEN** 참가자의 주 포지션에 빈자리가 있다
- **THEN** 시스템은 해당 참가자를 주 포지션에 우선 배정한다

#### Scenario: 가능 포지션으로 대체 배정
- **WHEN** 참가자의 주 포지션이 이미 채워져 있다
- **THEN** 시스템은 참가자의 가능 포지션 중 빈 포지션에 배정한다

#### Scenario: 포지션 배정 불가
- **WHEN** 10명의 포지션 조합으로 양 팀 5포지션을 모두 채울 수 없다
- **THEN** 시스템은 포지션 배정이 불가능하다는 메시지와 함께 문제가 되는 포지션을 표시한다

### Requirement: MMR 밸런스 매칭
시스템은 양 팀의 총 MMR 차이가 최소가 되도록 팀을 구성해야 한다.

#### Scenario: 밸런스 최적화
- **WHEN** 10명의 참가자가 포지션 배정 가능하다
- **THEN** 시스템은 가능한 팀 조합 중 양 팀 총 MMR 차이가 가장 작은 조합을 선택한다

#### Scenario: 후보 조합 제시
- **WHEN** 매칭 알고리즘이 완료된다
- **THEN** 시스템은 MMR 차이가 적은 상위 3개 팀 조합을 보여주고, 각 조합의 팀별 총 MMR과 차이를 표시한다

### Requirement: 매칭 결과 확인 및 확정
시스템은 매칭 결과를 보여주고 운영자가 선택하여 확정할 수 있어야 한다.

#### Scenario: 매칭 결과 표시
- **WHEN** 매칭 알고리즘이 후보 조합을 생성한다
- **THEN** 시스템은 각 후보에 대해 팀A/팀B의 멤버 이름, 배정 포지션, 개인 MMR, 팀 총 MMR을 표시한다

#### Scenario: 매칭 확정
- **WHEN** 운영자가 후보 조합 중 하나를 선택하고 확정한다
- **THEN** 시스템은 해당 조합으로 in_progress 상태의 매치를 서버에 생성하고, 전적 페이지로 이동한다

#### Scenario: 리롤
- **WHEN** 운영자가 제시된 후보가 마음에 들지 않아 리롤 버튼을 누른다
- **THEN** 시스템은 매칭 알고리즘을 다시 실행하여 새로운 후보 조합을 제시한다
