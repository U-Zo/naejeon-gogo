## MODIFIED Requirements

### Requirement: 멤버 슬롯 드래그앤드롭
시스템은 매칭 후보 카드 내 멤버 슬롯을 드래그하여 다른 슬롯 위치로 이동할 수 있어야 한다. 모바일 터치 환경에서 드래그 조작 시 브라우저 기본 스크롤이 발생하지 않아야 한다.

#### Scenario: 같은 팀 내 포지션 교환
- **WHEN** 운영자가 팀A의 멤버 슬롯을 드래그하여 같은 팀A의 다른 포지션 슬롯에 드롭한다
- **THEN** 시스템은 두 슬롯의 멤버와 포지션을 교환한다

#### Scenario: 팀 간 멤버 교환
- **WHEN** 운영자가 팀A의 멤버 슬롯을 드래그하여 팀B의 포지션 슬롯에 드롭한다
- **THEN** 시스템은 두 슬롯의 멤버를 교환하고, 각 멤버는 드롭된 슬롯의 포지션을 유지한다

#### Scenario: 드래그 시각적 피드백
- **WHEN** 운영자가 멤버 슬롯을 드래그하기 시작한다
- **THEN** 시스템은 드래그 중인 슬롯을 시각적으로 구분하고, 유효한 드롭 대상 슬롯을 강조 표시한다

#### Scenario: 카드 외부 드롭
- **WHEN** 운영자가 멤버 슬롯을 같은 후보 카드 외부에 드롭한다
- **THEN** 시스템은 드래그를 취소하고 원래 위치로 복원한다

#### Scenario: 모바일 터치 드래그 시 스크롤 방지
- **WHEN** 운영자가 모바일 환경에서 멤버 슬롯을 터치하여 드래그한다
- **THEN** 시스템은 해당 슬롯 영역에서 브라우저 기본 스크롤을 방지하고 드래그 동작만 수행한다

#### Scenario: 드래그 영역 외부 스크롤 유지
- **WHEN** 운영자가 모바일 환경에서 멤버 슬롯이 아닌 영역을 터치하여 스크롤한다
- **THEN** 시스템은 기존과 동일하게 페이지 스크롤을 허용한다
