## Why

현재 매칭메이킹 시스템은 알고리즘이 생성한 후보 조합을 그대로 선택하거나 리롤만 가능하다. 운영자가 특정 멤버의 팀이나 포지션을 미세 조정하고 싶을 때(예: 친목 목적, 특수 연습 매치, 밸런스 체감 조정) 매칭을 처음부터 다시 돌려야 하며, 원하는 결과가 나올 때까지 반복해야 한다. 드래그앤드롭으로 후보 조합을 직접 수정할 수 있으면 운영 효율이 크게 향상된다.

## What Changes

- 매칭 후보 카드에서 멤버를 드래그하여 같은 팀 내 다른 포지션으로 이동 가능
- 멤버를 드래그하여 상대 팀의 특정 포지션 슬롯으로 이동 가능 (팀 간 교환)
- 드래그앤드롭으로 수정 시 MMR 차이가 실시간으로 재계산되어 표시
- 수정된 조합은 시각적으로 "수정됨" 상태로 구분 표시
- 수정된 조합도 기존과 동일하게 확정(생성) 가능

## Capabilities

### New Capabilities
- `drag-drop-team-edit`: 매칭 후보 카드 내 멤버 슬롯의 드래그앤드롭 재배치 기능. 같은 팀 내 포지션 교환, 팀 간 멤버 교환, 실시간 MMR 재계산, 수정 상태 표시를 포함한다.

### Modified Capabilities
- `matchmaking`: 매칭 결과 확인 및 확정 요구사항에 "운영자가 후보 조합을 드래그앤드롭으로 수정한 뒤 확정할 수 있다"는 시나리오가 추가된다.

## Impact

- **프론트엔드**: `MatchmakingPage.tsx`의 CandidatesPhase에 드래그앤드롭 인터랙션 추가, `MatchmakingPage.css.ts`에 드래그 상태 스타일 추가
- **의존성**: 드래그앤드롭 라이브러리 추가 필요 (예: `@dnd-kit/core`)
- **데이터 모델**: `MatchCandidate` 타입은 변경 없음 — 프론트엔드 상태에서 수정본을 관리하고 기존 API로 확정
- **서버**: 변경 없음 — MMR 재계산은 클라이언트에서 수행 (멤버별 MMR 데이터는 이미 클라이언트에 존재)
