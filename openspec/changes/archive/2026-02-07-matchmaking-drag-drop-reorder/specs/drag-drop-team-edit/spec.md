## ADDED Requirements

### Requirement: 멤버 슬롯 드래그앤드롭
시스템은 매칭 후보 카드 내 멤버 슬롯을 드래그하여 다른 슬롯 위치로 이동할 수 있어야 한다.

#### Scenario: 같은 팀 내 포지션 교환
- **WHEN** 운영자가 팀A의 멤버 슬롯을 드래그하여 같은 팀A의 다른 포지션 슬롯에 드롭한다
- **THEN** 시스템은 두 슬롯의 멤버와 포지션을 교환한다

#### Scenario: 팀 간 멤버 교환
- **WHEN** 운영자가 팀A의 멤버 슬롯을 드래그하여 팀B의 포지션 슬롯에 드롭한다
- **THEN** 시스템은 두 슬롯의 멤버를 교환하고, 각 멤버는 드롭된 슬롯의 포지션을 유지한다

#### Scenario: 드래그 시각적 피드백
- **WHEN** 운영자가 멤버 슬롯을 드래그하기 시작한다
- **THEN** 시스템은 드래그 중인 슬롯을 시각적으로 구분하고, 유효한 드롭 대상 슬롯을 강조 표시한다

#### Scenario: 카드 외부 드롭
- **WHEN** 운영자가 멤버 슬롯을 같은 후보 카드 외부에 드롭한다
- **THEN** 시스템은 드래그를 취소하고 원래 위치로 복원한다

### Requirement: 실시간 MMR 재계산
시스템은 드래그앤드롭으로 멤버 배치가 변경될 때 MMR 합계와 차이를 즉시 재계산하여 표시해야 한다.

#### Scenario: 팀 내 교환 시 MMR 변화 없음
- **WHEN** 같은 팀 내에서 포지션 교환이 발생한다
- **THEN** 시스템은 양 팀의 총 MMR과 MMR 차이를 동일하게 유지한다

#### Scenario: 팀 간 교환 시 MMR 재계산
- **WHEN** 팀 간 멤버 교환이 발생한다
- **THEN** 시스템은 양 팀의 총 MMR을 재계산하고, MMR 차이 배지를 갱신한다

### Requirement: 수정 상태 표시
시스템은 원본 후보 조합과 비교하여 수정 여부를 시각적으로 표시해야 한다.

#### Scenario: 수정된 후보 카드 표시
- **WHEN** 후보 카드의 멤버 배치가 원본과 달라진다
- **THEN** 시스템은 해당 후보 카드에 "수정됨" 배지를 표시한다

#### Scenario: 원본 복원 시 수정 표시 제거
- **WHEN** 수정된 후보 카드의 멤버 배치가 다시 원본과 동일해진다
- **THEN** 시스템은 "수정됨" 배지를 제거한다

### Requirement: 드래그와 카드 선택 동작 분리
시스템은 드래그앤드롭 인터랙션과 후보 카드 선택 클릭을 구분해야 한다.

#### Scenario: 드래그 시 카드 선택 방지
- **WHEN** 운영자가 멤버 슬롯을 드래그한다
- **THEN** 시스템은 해당 동작을 카드 선택으로 처리하지 않는다

#### Scenario: 슬롯 외부 클릭으로 카드 선택
- **WHEN** 운영자가 멤버 슬롯이 아닌 카드 영역을 클릭한다
- **THEN** 시스템은 기존과 동일하게 해당 후보 카드를 선택한다
