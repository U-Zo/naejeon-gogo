## MODIFIED Requirements

### Requirement: 매칭 결과 확인 및 확정
시스템은 매칭 결과를 보여주고 운영자가 선택하여 확정할 수 있어야 한다. 운영자는 후보 조합을 드래그앤드롭으로 수정한 뒤 확정할 수도 있다.

#### Scenario: 매칭 결과 표시
- **WHEN** 매칭 알고리즘이 후보 조합을 생성한다
- **THEN** 시스템은 각 후보에 대해 팀A/팀B의 멤버 이름, 배정 포지션, 개인 MMR, 팀 총 MMR을 표시한다

#### Scenario: 매칭 확정
- **WHEN** 운영자가 후보 조합 중 하나를 선택하고 확정한다
- **THEN** 시스템은 해당 조합으로 in_progress 상태의 매치를 서버에 생성하고, 전적 페이지로 이동한다

#### Scenario: 수정된 매칭 확정
- **WHEN** 운영자가 드래그앤드롭으로 수정한 후보 조합을 선택하고 확정한다
- **THEN** 시스템은 수정된 조합의 teamA, teamB 데이터로 매치를 서버에 생성하고, 전적 페이지로 이동한다

#### Scenario: 리롤
- **WHEN** 운영자가 제시된 후보가 마음에 들지 않아 리롤 버튼을 누른다
- **THEN** 시스템은 매칭 알고리즘을 다시 실행하여 새로운 후보 조합을 제시하고, 이전 수정 사항은 초기화된다
