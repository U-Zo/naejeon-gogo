## Why

현재 매치메이킹은 밸런스 기반 알고리즘으로 팀을 구성하는 방식만 존재한다. 가끔은 밸런스 없이 순수 랜덤으로 팀을 짜고 싶을 때가 있고, 그 과정 자체를 시각적으로 재미있게 보여주면 내전 분위기를 살릴 수 있다. 달리기 경주 룰렛으로 "누가 어디 팀에 갈지"를 애니메이션으로 보여주는 새로운 팀 구성 방식을 추가한다.

## What Changes

- **새 페이지 추가**: 달리기 경주 룰렛 페이지 (`/race`) 신설
- **레이싱 애니메이션**: 선택된 10명의 멤버가 캐릭터로 등장, 왼쪽에서 오른쪽으로 달리는 경주 애니메이션
- **카메라 연출**: 경주 시작 시 전체 레이스를 조감하다가, 골인하는 멤버가 생길수록 남은 인원들에게 카메라가 점점 줌인되어 긴장감 연출
- **물리 엔진 기반 시뮬레이션**: 물리 엔진을 사용하여 레이서들의 가속·감속·충돌 등을 시뮬레이션. 매번 다른 전개가 나오는 예측 불가능한 경주 연출
- **경주 시간**: 한 경기당 최소 1분 ~ 최대 2분 정도의 길이. 너무 짧지 않게 긴장감을 유지하면서도 지루하지 않은 적정 시간
- **풀랜덤 팀 배정**: 서버 밸런스 계산 없이, 물리 시뮬레이션 결과로 순위 결정 → 상위 5명 팀A, 하위 5명 팀B
- **결과 확정 플로우**: 경주 종료 후 결과를 확인하고 기존 매치 생성 API로 확정 가능
- **기존 매칭 페이지 유지**: `/` 기존 밸런스 매칭은 그대로 유지, 네비게이션에서 선택 가능

## Capabilities

### New Capabilities
- `race-roulette`: 달리기 경주 애니메이션 기반 풀랜덤 팀 구성 페이지. 멤버 선택 → 경주 애니메이션 → 팀 배정 결과 → 매치 확정의 전체 플로우를 포함한다.

### Modified Capabilities
_(기존 스펙 변경 없음 — 새 페이지를 독립적으로 추가)_

## Impact

- **라우팅**: TanStack Router에 `/race` 라우트 추가
- **네비게이션**: Layout 컴포넌트에 경주 룰렛 링크 추가
- **클라이언트 코드**: `src/client/pages/race/` 디렉터리에 새 페이지 컴포넌트 및 애니메이션 로직 추가
- **서버 코드**: 변경 없음 (풀랜덤이므로 서버 밸런스 계산 불필요, 매치 확정 시 기존 `createMatch` API 재활용)
- **의존성**: 물리 엔진(matter.js 등) 도입하여 레이서 간 충돌·가속·감속 등 자연스러운 물리 시뮬레이션 구현, Canvas API로 렌더링
- **스타일링**: vanilla-extract로 레이싱 관련 스타일 추가
