## ADDED Requirements

### Requirement: 한글 초성 검색
시스템은 검색어가 한글 자음(초성)으로만 구성된 경우, 멤버 이름에서 초성을 추출하여 검색어와 매칭해야 한다(SHALL). 초성 매칭은 부분 문자열(substring) 방식으로 동작해야 한다(SHALL).

#### Scenario: 초성으로 멤버 검색
- **WHEN** 사용자가 검색창에 "ㄱㅊㅅ"를 입력한다
- **THEN** 시스템은 이름의 초성이 "ㄱㅊㅅ"를 포함하는 멤버(예: "김철수")를 검색 결과에 표시한다

#### Scenario: 초성 부분 매칭
- **WHEN** 사용자가 검색창에 "ㅊㅅ"를 입력한다
- **THEN** 시스템은 이름의 초성에 "ㅊㅅ"가 포함된 멤버(예: "김철수"의 초성 "ㄱㅊㅅ"에 "ㅊㅅ" 포함)를 검색 결과에 표시한다

#### Scenario: 단일 초성 검색
- **WHEN** 사용자가 검색창에 "ㄱ"을 입력한다
- **THEN** 시스템은 이름의 초성에 "ㄱ"이 포함된 모든 멤버를 검색 결과에 표시한다

#### Scenario: 초성이 아닌 일반 검색어
- **WHEN** 사용자가 검색창에 "김철"을 입력한다 (초성이 아닌 문자 포함)
- **THEN** 시스템은 기존 substring 매칭으로 검색한다 (초성 매칭을 사용하지 않는다)

#### Scenario: 매칭되는 멤버가 없는 경우
- **WHEN** 사용자가 검색창에 "ㅋㅋㅋ"를 입력하고 해당 초성에 매칭되는 멤버가 없다
- **THEN** 시스템은 빈 검색 결과를 표시한다

### Requirement: 검색 로직 공유
멤버 목록 페이지와 매치메이킹 페이지는 동일한 검색 로직(substring + 초성 매칭)을 사용해야 한다(SHALL).

#### Scenario: 멤버 목록에서 초성 검색
- **WHEN** 사용자가 멤버 관리 페이지의 검색창에 초성을 입력한다
- **THEN** 시스템은 초성 매칭으로 멤버를 필터링한다

#### Scenario: 매치메이킹에서 초성 검색
- **WHEN** 사용자가 매치메이킹 페이지의 검색창에 초성을 입력한다
- **THEN** 시스템은 초성 매칭으로 후보 멤버를 필터링한다
