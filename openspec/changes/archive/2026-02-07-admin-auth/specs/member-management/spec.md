## MODIFIED Requirements

### Requirement: 멤버 정보 수정
시스템은 등록된 멤버의 이름, 주 포지션, 가능 포지션, MMR을 수정할 수 있어야 한다. 멤버 수정은 관리자로 로그인한 경우에만 가능하다.

#### Scenario: 포지션 변경
- **WHEN** 관리자가 로그인한 상태에서 멤버의 주 포지션을 "mid"에서 "jungle"로 변경한다
- **THEN** 시스템은 해당 멤버의 주 포지션을 업데이트하고 저장한다

#### Scenario: MMR 수동 조정
- **WHEN** 관리자가 로그인한 상태에서 멤버의 MMR을 직접 수정한다
- **THEN** 시스템은 입력된 MMR 값으로 업데이트한다

#### Scenario: 비관리자 수정 버튼 숨김
- **WHEN** 사용자가 관리자로 로그인하지 않은 상태에서 멤버 목록을 본다
- **THEN** 멤버 카드의 수정 기능(카드 클릭 시 편집)이 비활성화된다

### Requirement: 멤버 삭제
시스템은 멤버를 삭제할 수 있어야 한다. 삭제 시 확인 절차를 거쳐야 한다. 멤버 삭제는 관리자로 로그인한 경우에만 가능하다.

#### Scenario: 멤버 삭제 확인
- **WHEN** 관리자가 로그인한 상태에서 멤버 삭제 버튼을 누른다
- **THEN** 시스템은 삭제 확인 다이얼로그를 표시한다

#### Scenario: 삭제 확정
- **WHEN** 관리자가 삭제 확인 다이얼로그에서 확정한다
- **THEN** 시스템은 해당 멤버를 목록에서 제거한다. 기존 전적 기록은 유지된다.

#### Scenario: 비관리자 삭제 버튼 숨김
- **WHEN** 사용자가 관리자로 로그인하지 않은 상태에서 멤버 목록을 본다
- **THEN** 멤버 카드의 삭제 버튼이 표시되지 않는다
