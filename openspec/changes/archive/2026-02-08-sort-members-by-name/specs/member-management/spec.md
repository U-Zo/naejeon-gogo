## MODIFIED Requirements

### Requirement: 멤버 목록 조회
시스템은 등록된 모든 멤버를 목록으로 표시해야 한다. 이름, 주 포지션, MMR, 전적(승/패)이 표시되어야 한다. 멤버 검색 시 기존 substring 매칭과 함께 한글 초성 매칭을 지원해야 한다(SHALL). 멤버 목록은 이름 기준 가나다순(오름차순)으로 정렬되어야 한다(SHALL).

#### Scenario: 멤버 목록 표시
- **WHEN** 운영자가 멤버관리 탭을 연다
- **THEN** 시스템은 모든 멤버를 이름 기준 가나다순으로 정렬하여 이름, 주 포지션, MMR, 승/패 전적과 함께 목록으로 표시한다

#### Scenario: 임시 참가자 구분
- **WHEN** 멤버 목록에 임시 참가자가 포함되어 있다
- **THEN** 시스템은 임시 참가자를 시각적으로 구분하여 표시한다

#### Scenario: 이름 substring 검색
- **WHEN** 사용자가 검색창에 일반 텍스트를 입력한다
- **THEN** 시스템은 이름에 해당 텍스트가 포함된 멤버를 필터링하여 표시한다

#### Scenario: 이름 초성 검색
- **WHEN** 사용자가 검색창에 한글 초성만으로 구성된 텍스트를 입력한다
- **THEN** 시스템은 이름의 초성에 해당 텍스트가 포함된 멤버를 필터링하여 표시한다

#### Scenario: API 응답 정렬
- **WHEN** 클라이언트가 멤버 목록 API를 호출한다
- **THEN** API는 멤버를 이름 기준 가나다순(오름차순)으로 정렬하여 반환한다
